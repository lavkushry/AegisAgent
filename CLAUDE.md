## What AegisAgent is (June 2026)

The **integrity layer for AI agent actions** â€” open, self-hostable, framework-neutral. The generic gateway loop (intercept â†’ policy â†’ allow/deny â†’ audit â†’ approval) is commodity (free Microsoft toolkit + OSS + SaaS), so it is **table stakes here**. The two defensible differentiators are:

1. **Approval integrity** â€” the human approval is bound to a SHA-256 hash of the *frozen exact action*; the SDK **fails closed** if a different/edited/expired action would execute (defeats approve-then-swap, replay, render-vs-bytes).
2. **Deterministic trust-provenance gating** â€” authorization is gated on the *source trust level* of the triggering content (6 levels), not a text score (confused-deputy defense). Plus **verifiable, hash-chained action receipts** as compliance evidence (SOC 2 / EU AI Act Art. 14).

> Motto: **Make the approval trustworthy. Trust the source, not the text.**

## Critical invariants (do not weaken)

- **Canonicalization `aegis-jcs-1` MUST stay byte-identical across SDK and gateway** (keys sorted by Unicode code point, compact separators, **raw UTF-8 / no `\uXXXX`**, reject non-finite floats). Locked by `tests/canonical_action_vectors.json` + `tests/receipt_chain_vectors.json`. A divergence silently breaks the fail-closed guarantee â€” never change hashing without bumping the scheme + CI byte-equality.
- **Fail closed:** unknown agent/tool/MCP server/MCP tool â†’ deny; critical â†’ deny; high-risk â†’ require approval. SDK refuses to execute on hash mismatch, expired approval, or unreachable gateway (mutating/high-risk).
- **Approval integrity:** every approval binds to the original `action_hash`; edits re-hash + re-evaluate; expiry enforced (SDK + gateway); never re-decide a decided approval; **single-use** â€” atomically consumed before execution (no replay).
- **Trust-provenance is deterministic:** classifiers may only *tighten* a label, never loosen it. Mutating action + `untrusted_external`/`malicious_suspected` â†’ deny. This tighten-only rule also governs cross-agent chains (#1293, `trust_chain::propagate`): a downstream hop's effective trust is gated on the most restrictive level seen anywhere upstream, never its own (possibly more trusting) self-report.
- **Multi-tenant isolation:** every tenant-owned query binds/filters `tenant_id`; parameterized SQLx only (no string interpolation).
- **Local binding** `127.0.0.1` for dev/test; **redact** secrets from logs/receipts (store hashes, not payloads); no `.unwrap()`/`.expect()` in production paths.
